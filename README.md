## Технические особенности

- Используется модель решающего дерева, обученная на исходных данных с искусственным добавлением шума.
- Полученное дерево автоматически переводится в код на С с помощью библиотеки `m2cgen`.
- На контроллере запускается только код на С, что обеспечивает минимальное потребление ресурсов.
- Для инференса используется только статическое выделение памяти с предварительной загрузкой обрабатываемого массива данных.

## ML-пайплайн

#### 1. Загрузка и подготовка данных 
Исходные данные не содержат аномалий, поэтому мы искусственно "портим" сигнал случайно выбранных датчиков. Добавляются одиночные скачки (на 5–15% от уровня сигнала) и блоки шума (гауссовский шум σ=5%). Это имитирует реальные аномалии в работе датчиков.

#### 2. Разбиение на train/val/test
Файлы `data/1, 2, 4, 5.csv` используются в обучающей выборке. Каждый из файлов содержит примерно по 23 потока данных, которые затем конкатенируются в один даатсет. Файл `data/3.csv` содержит 40 потоков данных, поэтому из него поровну распределяем потоки для валидации и тестирования с сохранением баланса числа "испорченных" датчиков.

#### 3. Извлечение признаков
Для каждой точки сигнала были вычислены: `rel_diff`(сглаженная абсолютная разность с предыдущей точкой, нормированная на скользящую медиану) и `rel_std` (скользящее стандартное отклонение, нормированное на ту же медиану). Нормировка на медиану делает признаки сопоставимыми для датчиков с разными физическими величинами и уровнями. Размер скользящего окна регулировался при обучении.

#### 4. Обучение модели
Была выбрана модель `DecisionTreeClassifier` из библиотеки `sklearn`, так как она легко конвертируется в набор правил `if-else`, которые будут очень быстро работать на контроллере с ограниченными ресурсами. В процессе валидации осуществлялся перебор гиперпараметров обучения, таких как `window_size` (размер скользящего окна для вычисления признаков), `corruption_ratio` (Доля датчиков, сигнал которых портим при обучении) и `max_depth` (максимальная глубина решающего дерева). В результате мы сохранили лучшую модель по метрике `f1` на валидационной выборке.

#### 5. Тестирование
Для финального тестированя использовалась отдельная тестовая выборка, которую модель никогда не видела в процессе обучения. Для каждого потока отдельно были посчитаны `accuracy`, `precision`, `recall` и `f1`, которые затем были агрегированы в средние значения по всем потокам.


## Полученные результаты

### Финальные метрики модели

- `mean accuracy` **= 0.99**
- `mean precision` **= 0.30**
- `mean recall` **= 0.97**
- `mean f1` **= 0.45**

В результатае получилась модель, которая реагирует почти на все аномалии, но дает много ложных срабатываний. Это может быть связано с тем, что для обучения и валидации использовались данные с разных типов промышленных датчиков. Учитывая это, можно ожидать. что опри реальной эксплуатации модель будет показывать себя примерно таким же образом, так как она не привязана к конкретным видам сигналов.

## Структура проекта

- [`diagnostic_module`](main) - скомпилированный код, запускается на контроллере.
- [`main.c`](main.c) - основной код для инференса модели.
- [`model.c`](model.c) - код ML-модели, генерируется автоматически по обученному дереву решений.
- [`crossplatform_main.c`](crossplatform_main.c) - копия основного кода на С, работающая как на Linux, так и на Windows. Подсчитывает затрачиваемые на выполнение ресурсы.
- `/input_data` - папка для входных CSV-файлов, программа автоматически берет первый файл.
- `/inference-reports` -  папка, куда сохраняются результаты по каждому датчику.
- [`/model_training/training.ipynb`](/model_training/training.ipynb) - код для обучения и валидации модели (Python).
- [`/model_training/requirements.txt`](/model_training/requirements.txt) - список зависимостей, необходимых для обучения модели.

## Инструкция по запуску

1. Установите wget с помощью команды (если не установлен):
```bash
apt install wget
```
2. Скачайте и запустите установочный скрипт с помощью команды:
```bash
wget https://raw.githubusercontent.com/rodomanovt/hack_gazprom/main/install.sh && chmod +x install.sh && ./install.sh
```
3. Перейдите в папку `/diagnostic_module`
```bash
cd diagnostic_module
```
4. Поместите CSV-файл с данными для анализа в папку `/input_data` (если он не был скачан при установке).
5. Для запуска программы используйте команду:
```bash
./diagnostic_module
```
6. Для каждого датчика будет сформирован CSV-отчет в папке `/inference_reports`.
